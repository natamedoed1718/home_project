## Использование
Проект предоставляет функции для работы с банковскими операциями:
- filter_by_state(data, state='EXECUTED') — фильтрует операции по статусу.

- sort_by_date(data, reverse=True) — сортирует операции по дате.

- mask_account_card(value) — маскирует номера карт и счетов.

- get_date(value) — форматирует дату для отображения.

Пример использования:

```
def filter_by_state(data: list, state: str = "EXECUTED") -> list:
    """Функция возвращает новый список словарей, содержащий только те словари, у которых ключ
    state соответствует указанному значению.
    """

    result = []

    for item in data:

        # проверяем, есть ли ключ 'state' и совпадает ли его значение
        if 'state' in item:
            if item['state'] == state:
                result.append(item)
    return result
```

## Новая функциональность: декоратор log

В проект добавлен декоратор log для автоматического логирования выполнения функций.

### Особенности:

* Логирование успешного выполнения функции с указанием её имени.

* Логирование ошибок, включая тип ошибки и входные параметры функции.

* Возможность логировать в файл или выводить в консоль:

* Если аргумент filename указан - лог пишется в файл.

* Если filename не указан - лог выводится в консоль.

* Декоратор типизирован, совместим с mypy и поддерживает любые функции с произвольными аргументами (*args, **kwargs).

### Логирование в файл
@log(filename="mylog.txt")
def my_function(x: int, y: int) -> int:
    return x + y

my_function(1, 2)  # В файл mylog.txt будет записано: "my_function ok"

### Логирование в консоль
@log()
def divide(a: float, b: float) -> float:
    return a / b

try:
    divide(1, 0)
except ZeroDivisionError:
    print("Произошла ошибка: деление на ноль")
#### В консоль будет выведено:
 "divide error: ZeroDivisionError. Inputs: (1, 0), {}"

### Преимущества:

Удобно для отладки и мониторинга работы функций.

Легко подключить к любым существующим функциям.

Универсальный и безопасный для использования в проекте.
